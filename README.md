# Hive Lovable UI

Starter template for Hive internal tools built with Lovable.

This repository provides a shared UI foundation so that apps generated by Lovable follow a consistent design system.

The goal is to prevent every generated tool from inventing its own UI patterns.

---

# Stack

This template is built with:

- Next.js (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- Geist font

All UI components inherit tokens defined in `globals.css`.

---

# Design Tokens

The UI tokens in this repository are mapped from the Hive design system.

Only semantic tokens are exposed to the UI layer so generated apps remain consistent.

---

### Primary action

Used for primary buttons and important actions.


background: #000000
text: #ffffff

primary surface: #ffffff

---

# Radius


small: 6px
medium: 8px (default)
large: 200px (pill)


Most components should use **8px** radius.

Use **200px** only for pill elements such as badges or chips.

---

# Spacing

Spacing scale:


4
8
16
24
32
40
48
56


Recommended usage:


page padding → p-6
card padding → p-4
grid gap → gap-4
section gap → mt-6
large spacing → gap-8


Avoid introducing custom spacing values.

---

# Typography

Primary UI font:


Geist


Available tokens:


font-sans → Geist
font-mono → Geist Mono


Do not introduce additional font families.

---

# Layout Structure

All pages should follow the shared layout pattern:


Sidebar
Header
Content


Content is typically structured as:


Page
└ Section
└ Card / Panel


Reuse existing layout patterns whenever possible.

---

# Components

Prefer using existing components from:


/components/ui


Common components:

- Button
- Card
- Input
- Table
- Select
- Dialog
- Badge
- Dropdown Menu
- Tabs

Avoid creating duplicate UI components unless necessary.

---

# Lovable Usage

When generating new features with Lovable:

- extend the existing project
- reuse the sidebar and page layout
- use shadcn components
- follow spacing tokens
- avoid introducing new design styles

Recommended prompt example:


Extend this project using the existing layout and components.
Follow the design tokens defined in globals.css.
Reuse spacing patterns and avoid introducing new UI styles.


---

# Contribution

If improving the UI foundation:

- keep changes generic
- align with the Hive design system
- avoid introducing product-specific styling

This repository should remain a **neutral starter for internal tools**.
